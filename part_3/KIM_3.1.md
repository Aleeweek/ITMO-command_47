# КИМ 3.1. Лабораторная работа (Интеграция PostgreSQL ↔ HDFS: экспорт, импорт, контроль форматов)

Тип оценочного средства (предмет контроля): защита отчета

Перечень контролируемых тем (модулей): Модуль 3

### Цель 
настроить и продемонстрировать способы движения табличных данных между PostgreSQL и HDFS, оценить потери/изменения формата, настроить проверки.

## Задачи
1. Подготовка таблицы в PostgreSQL (пример: транзакции/логи).
2. Экспорт таблицы в CSV (pg_dump/copy to) и загрузка в HDFS.
3. Конвертация CSV → Parquet (используя Spark/pyarrow).
4. Импорт Parquet обратно в PostgreSQL (через staging + COPY или через Spark JDBC).
5. Сравнение исходного и восстановленного набора (row count, checksum выборки).
6. Оформление отчёта с выводами о потерях/изменении типов.
   
## Варианты задания
1. Экспортировать таблицу с BLOB-полем и организовать хранение BLOB отдельно.
2. Экспортировать с разностью кодировок (UTF-8 vs Latin1) и исправить при импорте.
3. Конвертация CSV→Parquet с сохранением метаданных (schema).
4. Использовать sqoop (если доступен) для переноса и оценить.
5. Выполнить bulk-load в PostgreSQL через COPY из локального файла, с проверкой транзакций.
6. Импортировать в PostgreSQL с преобразованием типов (string→timestamp).
7. Оценить скорость при разных размерах batch при импорте.
8. Сравнить результат при использовании разных парсеров (pandas vs pyarrow).
9. Реализовать автоматическую проверку схемы (schema registry-like).
10. Реализовать этап валидации и отката при несоответствии количества строк.
