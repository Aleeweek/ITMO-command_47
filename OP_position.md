# Позиция дисциплины в образовательной программе для подготовки специалистов в области ИИ

## Предшествующие дисциплины

Изучение дисциплины базируется на знаниях, полученных обучающимися при освоении следующих дисциплин:
* дискретная математика;
* алгоритмизация и программирование;
* проектирование баз данных.
 
На основе которых студент должен иметь следующие **знания**:
* основные понятия дискретной математики по темам: теория множеств, теория отношений и теории графов;
* основы реляционной алгебры (операции над отношениями);
* базовые принципы алгоритмизации (линейные, ветвящиеся и циклические структуры);
* основные структуры данных (массивы, записи, деревья);
* этапы проектирования пользовательских приложений;
* базовые понятия реляционной модели данных (отношение, атрибут, кортеж, домен, первичный и внешний ключи), а также формальные основы реляционной алгебры (операции выборки, проекции, соединения, деления) и реляционного исчисления (кортежные и доменные переменные);
* классические этапы проектирования баз данных (концептуальное, логическое, физическое) и методологию построения диаграмм «сущность-связь» (ER-модель) с выделением сущностей, атрибутов и связей различной кратности;
* правила преобразования ER-схемы в реляционную структуру с сохранением семантики предметной области;
* теорию функциональных зависимостей, нормальные формы (1НФ, 2НФ, 3НФ, НФБК) и алгоритмы декомпозиции отношений для устранения аномалий модификации и избыточности данных.

**Умения**:
* корректно применять основные алгоритмы для выполнения реляционных операций над отношениями;
* решать задачи математического программирования на основе аналитических методов;
* формализовать предметную область с использованием математических моделей;
* разрабатывать алгоритмы средней сложности на языке программирования;
* работать с составными типами данных (структуры, объекты);
* анализировать вычислительную сложность алгоритмов;
* строить концептуальную модель предметной области в нотации ER-диаграммы по текстовому описанию задачи, выполнять ее логическое преобразование в набор правильно нормализованных реляционных таблиц с обоснованным выбором первичных и внешних ключей;
* применять аппарат реляционной алгебры для формализации запросов любой сложности (включая соединения нескольких отношений, группировки, подзапросы) и транслировать эти формальные выражения в синтаксически корректные SQL-запросы;
* анализировать заданную схему базы данных на предмет наличия избыточности и аномалий, выполнять проверку на соответствие требованиям нормализации и при необходимости проводить декомпозицию отношений без потерь данных и с сохранением функциональных зависимостей.

**Навыки**:
* работы с современными средствами проектирования пользовательских приложений;
* решения логических задач из дискретной математики;
* программирования на одном из языков;
* обработки структур данных в программах;
* проверки схемы данных на нормализацию с использованием аппарата замыканий атрибутов и неприводимых покрытий множества функциональных зависимостей;
* документирования результатов проектирования в виде спецификации схемы данных (описание таблиц, атрибутов, ограничений, индексов);
* устранения типовых ошибок проектирования (избыточность, несогласованность, неполнота модели) на основе анализа семантики предметной области.
 
## Кореквизиты (смежные) дисциплины. Освоение данной дисциплины проходит параллельно со следующими дисциплинами:
* технологии проектной деятельности;
* технологическая практика.
 
## Постреквизиты дисциплины. Освоение данной дисциплины является теоретической и практической базой для следующих дисциплин и практик:
* практикум по цифровому проектированию;
* большие данные и облачные технологии;
* управление информационными ресурсами;
* проектно-технологическая практика;
* архитектор данных;
* выполнение и защита ВКВ.
